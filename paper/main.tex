\begin{abstract}
KV cache compression reduces autoregressive inference memory, but aggressive pruning often trades quality for speed. We study this tradeoff on EleutherAI/Pythia-70M with NVIDIA KVPress, no training, and fp32 eager attention. Beyond Dense, StreamingLLM, SnapKV, PyramidKV, ExpectedAttention, ObservedAttention, and TOVA baselines, we iteratively test hybrid and layer-budget variants. The selected aggressive-compression method is Expected Soft Pyramid: it keeps ExpectedAttention as the token scorer and applies a mild low-to-high layer budget schedule. At a 132-token target cache for a 512-token context, it lies on the Pareto frontier and has the best equal-rank balance over average perplexity, TPOT, and decode-attention FLOPs. We also report that adding observed-attention residuals often hurts WikiText quality, which makes the conservative ExpectedAttention-based variant the most reliable innovation.
\end{abstract}

\section{Introduction}

The KV cache stores one key and value vector per layer and token, so long-context decoding stresses GPU memory and attention bandwidth. The assignment asks us to combine ideas from four levels: attention mechanisms, per-layer KV eviction, inter-layer cache allocation, and macro inference architecture. We keep the model fixed, use no training, and use KVPress~\citep{kvpress} to hook Transformers during prefill.

Pythia-70M~\citep{biderman2023pythia} uses GPT-NeoX attention modules and partial RoPE, while several KVPress utilities assume LLaMA-like module names and full-dimensional RoPE. Our adapter only supplies these structural aliases and the partial-RoPE correction needed for ExpectedAttention. All compression decisions are implemented as KVPress-style presses that gather existing KV tensors along the sequence dimension.

Expected Soft Pyramid uses the ExpectedAttention score from KVPress but changes the budget per layer:
\[
B_\ell=\mathrm{round}(B\alpha_\ell),\quad
\alpha_\ell\in[1.25,0.75],
\]
where $\alpha_\ell$ decreases linearly from lower to higher layers and averages to one. This keeps the global target cache size unchanged while testing the common observation that lower layers can benefit from broader lexical context. Sink tokens are always protected. We also test Hybrid Expected-Observed,
\[
s = 0.55\,\hat{s}_{\mathrm{expected}} + 0.30\,\hat{s}_{\mathrm{observed}} + 0.15\,\hat{s}_{\mathrm{vnorm}},
\]
where each score is min-max normalized per batch and head. Sink tokens and a 32-token recent window receive a protection boost before top-$k$ selection. Residual and soft-pyramid hybrid variants are retained as ablations.

\section{Experiments}

We evaluate `EleutherAI/pythia-70m' on WikiText-2 test~\citep{merity2017wikitext} and one PG-19 test stream sample~\citep{rae2019compressive}. For PPL, a 512-token context is prefetched and compressed before scoring the next 128 continuation tokens. For generation, a 512-token PG-19 prompt generates 32 tokens. The decode attention FLOPs proxy is $\sum_{\ell,t}4HDL_{\ell,t}$. Reported runs use an NVIDIA GeForce RTX 5060 Laptop GPU.

We first screen new baselines and hybrids on 256-token contexts and target cache sizes 96, 128, and 192. \section{Analysis and Limitations}

The main result is not that a more complicated scorer always wins. ExpectedAttention is clearly the quality anchor, especially on WikiText-2. The successful innovation is conservative: Expected Soft Pyramid changes the inter-layer budget while keeping the ExpectedAttention scorer intact. It gives up about 2 average PPL relative to ExpectedAttention at target 132, but improves TPOT and remains on the Pareto frontier. In contrast, adding observed-attention residuals improves some PG-19 cases but tends to damage WikiText quality, so we report those variants as negative ablations rather than the recommended method.

The evaluation is intentionally small and reproducible: one model, no training, two datasets, fixed seeds, and one GPU. Timing differences are small at Pythia-70M scale, so the paper reports exact CSVs rather than over-claiming hardware speedups. We do not implement MQA/GQA/MLA or YOCO-style cross-layer sharing because those require architecture or weight changes outside the no-training setting. KVPress SimLayerKV is excluded because its full-RoPE assumption is incompatible with GPT-NeoX partial RoPE without further changes.
